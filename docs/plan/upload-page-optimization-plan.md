# UploadPage 上传页面优化方案

## 一、现状分析

### 当前页面结构
- 两栏布局：左侧拖拽上传区 + 右侧粘贴文本区
- 底部文件列表（平铺展示，无分类）
- 可展开的 Markdown 预览面板
- 每个文件单独的「摄取」按钮

### 存在的问题

| 问题 | 严重程度 | 描述 |
|------|----------|------|
| 布局单调 | 高 | 简单的 2-grid + list，缺乏视觉层次和信息密度 |
| 功能单一 | 高 | 只能上传文件/粘贴文本 + 逐个摄取，无批量操作 |
| 缺少删除 | 高 | 界面和后端均不支持删除文件 |
| 无搜索筛选 | 中 | 文件多时无法快速定位 |
| 无进度反馈 | 中 | 上传大文件时无进度条，体验不好 |
| 无文件分类 | 中 | 所有类型文件混在一起，无类型 Tab 区分 |
| 无批量操作 | 中 | 不能批量选择 + 批量摄取 |
| 预览简陋 | 低 | 预览面板缺乏元数据展示和交互 |
| 状态缺失 | 低 | 不知道哪些文件已摄取、哪些未处理 |
| 空状态单调 | 低 | 无文件时只有文字提示，无引导性视觉 |

### 3.2 新增功能清单

| 功能 | 优先级 | 描述 |
|------|--------|------|
| **统计卡片** | P0 | 顶部展示文件总数、总大小、已摄取数量 |
| **上传进度条** | P0 | 上传文件时显示实时进度（XMLHttpRequest + onprogress） |
| **文件类型图标** | P0 | 根据扩展名显示彩色图标（.md=蓝、.pdf=红、.xlsx=绿 等） |
| **搜索筛选** | P0 | 在文件列表顶部添加搜索框，按文件名实时过滤 |
| **类型 Tab** | P1 | 全部 / 文档 / 表格 / 其他 四种类型筛选 |
| **排序选项** | P1 | 最新 / 名称 / 大小 三种排序方式 |
| **批量选择** | P1 | 复选框多选，全选/取消全选 |
| **批量摄取** | P1 | 选中多个文件后一键批量摄取 |
| **删除文件** | P0 | 单文件删除 + 批量删除，带确认弹窗 |
| **摄取状态** | P1 | 标记文件是否已摄取（对比 wiki/sources/ 目录） |
| **增强预览** | P1 | 预览面板含元数据（文件大小、修改时间、类型、转换状态） |
| **格式转换提示** | P1 | 上传非 MD 文件后显示「已自动转换为 Markdown」标签 |
| **快捷操作** | P2 | 点击文件名直接预览，拖拽时显示视觉反馈 |
| **空状态引导** | P2 | 无文件时显示插画 + 「拖拽文件开始」引导文字 |
| **Toast 优化** | P2 | 优化 Toast 样式和动画位置 |

### 3.3 后端新增 API

| 端点 | 方法 | 描述 |
|------|------|------|
| `/api/raw-files/{path}` | DELETE | 删除指定文件（仅限 raw/ 目录内） |
| `/api/raw-file-content` | GET | 获取文件原始文本内容用于预览（已有） |
| `/api/ingest-status` | GET | 检查文件是否已被摄取（可选，对比 sources/ 目录） |

### 3.4 文件类型配色方案

```
.md/.txt     → 蓝色系  (#0a84ff)  📄 FileText
.pdf         → 红色系  (#ff453a)  📕 FileType
.docx/.rtf   → 靛蓝系  (#5856d6)  📝 FileEdit
.xlsx/.csv   → 绿色系  (#30d158)  📊 Table2
.pptx        → 橙色系  (#ff9f0a)  📽 Presentation
.html/.xml   → 紫色系  (#bf5af2)  🌐 Code2
.json/.yaml  → 青色系  (#64d2ff)  🔧 Braces
图片/音频    → 粉色系  (#ff375f)  🎵 FileAudio
其他         → 灰色系  (#98989d)  📎 Paperclip
```

### 3.5 交互细节

1. **拖拽上传时**：拖拽区域变蓝 + 边框加粗 + 显示「松开鼠标上传文件」
2. **上传进行中**：文件列表顶部出现进度条 + 文件名旁显示 spinner
3. **批量选择后**：底部浮现操作栏（slide-up 动画），显示已选数量和操作按钮
4. **删除确认**：弹出确认对话框，明确告知不可撤销
5. **摄取完成后**：文件旁状态图标从 ⏳ 变为 ✅，toast 显示成功
6. **搜索时**：实时过滤 + 空结果提示

### 文件状态判断逻辑

```typescript
// 通过文件路径判断是否已摄取：检查 wiki/sources/ 中是否有对应页面
// 前端通过预览 API 的 200/404 结果判断
async function checkIngestedStatus(path: string): Promise<boolean> {
  const slug = path.replace(/\.(md|txt)$/, '').split('/').pop();
  try {
    const res = await fetch(`/api/pages/sources/${slug}`);
    return res.ok;
  } catch {
    return false;
  }
}
```
