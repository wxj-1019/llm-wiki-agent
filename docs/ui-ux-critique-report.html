LLM Wiki Viewer · UI/UX Critique

整体完成度较高的 Apple 设计语言实现，功能性扎实，但缺少统一的设计哲学锚点和差异化记忆点，视觉层级在复杂页面上有竞争。

Overall
1
哲学一致性 Strong
Philosophy Consistency · 7 / 10
全局设计系统（index.css）建立了完整的 Apple 设计语言：统一的 --apple-* 色彩变量、glass morphism（.glass + backdrop-filter: blur(20px)）、apple-card / apple-button 组件体系。圆角半径（rounded-2xl = 16px、rounded-xl = 12px）在所有页面保持一致。深浅色主题通过 CSS 变量无缝切换。Marshmallow 色系（mint/blush/cream/sky/lavender/peach）作为辅助调色盘统一使用。

不一致点：SettingsPage 的 Tab 使用 bg-[var(--bg-secondary)] 分段控制器，而 BrowsePage 的 Tab 使用独立药丸样式（无统一容器背景）。GraphPage 底部控制栏的密度和布局模式与其他页面差异较大。UploadPage 的 stats 卡片用 bg-apple-blue/8，Dashboard 用 bg-apple-blue/10，opacity 不统一。
Keep Apple 设计 token 体系 + glass morphism + marshmallow 色板是很好的方向，不要动摇
Fix 统一所有页面 Tab 组件的容器样式（选 segmented control 或 pill 二选一）
QW 将 /8 → /10 和 font-bold → font-semibold 全局对齐，5 分钟搞定
2
视觉层级 Functional
Visual Hierarchy · 6 / 10
每个页面有清晰的 h1（text-3xl font-semibold）作为主锚点。HomePage 的时间问候语 → 搜索框 → 统计卡片 → 内容区域的阅读动线合理。PageDetailPage 的面包屑 → 标题 → 类型徽章 → 内容层级清晰。

问题：BrowsePage 的卡片网格中，每张卡片视觉权重相同——没有"重要/最近更新"的差异化呈现，用户无法快速区分优先级。DashboardPage 有 8 张统计卡片（两行 × 四列），视觉噪音大，核心指标（Health Score）被淹没在同等大小的卡片中。GraphPage 的 NodePanel（右侧详情）与底部控制栏同时出现时，信息密度超出舒适区。
Keep h1 标题 + 语义色彩 + 渐进入场动画的三层层级框架
Fix Dashboard 合并为 4 张核心卡片，将 secondary stats 下沉为小型 inline 指标
Fix BrowsePage 卡片增加"最近更新"高亮（如左边框色条或角标），区分新鲜度
QW Dashboard Health Score 卡片加粗/放大字号，使其成为视觉焦点
3
细节执行 Strong
Detail Execution · 7 / 10
Glass morphism 实现精致——backdrop-filter: blur(20px) 搭配半透明背景和细边框在浅色/深色下都表现良好。按钮的 hover（translateY(-1px)）、focus-visible（蓝色描边 + offset）、active 状态都经过设计。自定义滚动条（6px 宽、var(--text-tertiary) 颜色）匹配主题。Markdown 渲染的排版（.markdown-body）覆盖了标题、引用、代码块、表格等完整样式。

粗糙处：GraphPage 底部控制栏在移动端 gap-1 使按钮间距过于紧凑（<1px 视觉间距）。UploadPage stats 的 icon 背景色使用 bg-[var(--bg-primary)]，而 Dashboard 用 bg-apple-blue/10 模式——同一设计意图，两种实现方式。ChatPage 的 session dropdown 的删除按钮只有 opacity-0 group-hover:opacity-100，但父元素没有 group class，导致删除按钮永远不可见（需修复）。prefers-reduced-motion 只在 .ambient-blob 上做了适配，其他 framer-motion 动画未降级。
Keep Glass morphism + 自定义滚动条 + Markdown 排版的细节水准
Fix ChatPage session 列表项加 group class 使删除按钮可见
Fix 统一 icon 背景色模式：全部用 bg-apple-{color}/10
QW 为 framer-motion 动画添加 prefers-reduced-motion 全局降级
4
功能性 Strong
Functionality · 8 / 10
功能完整性是最大亮点：搜索框支持键盘导航（↑↓ Enter Escape），ChatPage 支持 SSE 流式对话 + 多会话管理 + 导出 Markdown，UploadPage 支持拖放/粘贴/批量操作/进度条，PageDetailPage 支持编辑/预览/分栏 + TOC 移动端抽屉。Focus trap（模态对话框）、body scroll lock、skip link 等无障碍措施到位。

不足：GraphPage 的 vis.js 在移动设备上无法正常缩放/拖拽（触控手势被页面滚动劫持）。社区筛选下拉缺少键盘导航（Tab/Enter）。BrowsePage 的排序下拉用原生 select，与整体设计语言不协调。
Keep 搜索键盘导航、Chat 流式、Upload 批量操作——这些是核心竞争力
Fix GraphPage 移动端添加 touch-action: none 防止手势冲突
QW BrowsePage 排序下拉替换为自定义 Select 组件（风格统一）
5
创新性 Functional
Innovation · 6 / 10
知识图谱 + 社区检测（Louvain 算法）的可视化是差异化亮点。Dashboard 的五维雷达图（Completeness/Consistency/Connectivity/Freshness/Diversity）比常见 dashboard 的纯数字展示更有洞察力。HomePage 的"随机发现"卡片和"继续阅读"功能提升了内容发现的趣味性。ChatPage 的"关于此页面提问"从详情页跳转并携带上下文参数，是很好的跨功能串联。

缺失：视觉语言完全在 Apple 设计手册的安全区内——没有一个"签名式"元素让人一眼记住这是 LLM Wiki Viewer。背景的 ambient blob 动画只在 HomePage 出现，其他页面回归纯色背景，丢失了氛围感。没有 micro-interaction 的惊喜（如收藏时的心跳动画已有但太短暂）。
Keep 雷达图 + 知识图谱社区 + 随机发现 + 跨页面上下文提问
Fix 设计一个可复用的品牌元素（如知识连线的抽象图形）应用到空白状态和加载态
QW Ambient blob 动画扩展到所有页面的 body::before，统一氛围

✅ Keep — 不要动的东西

  • Apple 设计 token 体系（--apple-* 色彩 + --bg/text/border 语义变量）+ 深浅色双主题——这是架构级决策，改了会全盘崩
  • Glass morphism 组件（.glass, .apple-card）的 backdrop-filter + border + shadow 三角组合
  • 搜索框的键盘导航模式（↑↓/Enter/Escape）+ 搜索结果分类标签
  • ChatPage 的 SSE 流式对话 + 多会话持久化 + Markdown 导出
  • PageDetailPage 的"关于此页面提问"跳转 Chat 带 context 参数——跨功能串联典范
  • Dashboard 五维健康雷达图——比纯数字 dashboard 有洞察力得多

🔧 Fix — 必须修复（按视觉收益/时间成本排序）

  • P0 ChatPage session 列表的删除按钮（X）因父元素缺少 group class 永远不可见——用户无法删除会话 P0
  • P0 Dashboard 8 张统计卡片视觉噪音过大——合并为 4 张核心卡（Sources/Entities/Concepts/Health Score），其余下沉为 inline 指标 P0
  • P1 BrowsePage 卡片无差异化——所有卡片等权等色，增加"最近 7 天更新"左边框色条或角标区分新鲜度 P1
  • P1 Tab 组件样式不统一——SettingsPage 用 segmented control，BrowsePage 用 pill tabs，选其一全局复用 P1
  • P1 GraphPage 移动端触控手势被页面滚动劫持——canvas 容器加 touch-action: none + 手势说明 P1
  • P1 prefers-reduced-motion 只覆盖了 ambient blob，framer-motion 的入场/过渡动画未降级——对前庭障碍用户是障碍 P1
  • P1 Icon 背景色模式不统一（bg-[var(--bg-primary)] vs bg-apple-blue/10）——全局对齐为后者 P1

⚡ Quick Wins — 5-15 分钟，收益大

  • bg-apple-blue/8 → /10，font-bold → font-semibold——全局搜索替换，统一视觉密度 5min
  • Ambient blob 动画从 HomePage 提取为 body::before 伪元素，所有页面共享氛围背景 10min
  • BrowsePage 排序下拉替换为自定义 <select> 组件（与 apple-input 风格匹配） 15min
  • GraphPage NodePanel 增加键盘 Escape 关闭 + 底部控制栏移动端 gap-2（当前 gap-1 太紧） 5min
  • Dashboard Health Score 卡片字号从 text-2xl 改为 text-3xl，颜色加粗——让它成为视觉锚点 3min
