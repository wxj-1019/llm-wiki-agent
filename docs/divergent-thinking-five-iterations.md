# LLM Wiki Agent — 五轮发散思维迭代报告

> 版本：v1.0 | 日期：2026-05-04 | 方法：每轮在前一轮基础上跨界×10

## 方法论：五轮发散框架

```
Round 1 ──► Round 2 ──► Round 3 ──► Round 4 ──► Round 5
竞品盲区    跨界启发      极端场景      技术前瞻      元设计
(已知边界)  (相邻领域)    (边缘案例)    (未来范式)    (终极形态)
```

每轮遵循「SCAMPER」+「第一性原理」+「反共识」三重思维模型：
- **SCAMPER**：替代(Substitute)、组合(Combine)、调整(Adapt)、修改(Modify)、另作他用(Put to other uses)、消除(Eliminate)、反转(Reverse)
- **第一性原理**：剥离既有假设，回到知识管理的本质
- **反共识**：主动寻找"所有人都认为不对"的方向

### 1.3 空白区 A 深度：知识变现层

**核心问题**：用户花了时间、精力、API 费用构建 Wiki，但 Wiki 本身不产生任何经济价值。

**竞品盲区**：
- Karpathy 原始 Gist 完全是个人工具思维
- MeMex 强调"零基础设施"但忽略了"零回报"
- casper 的 Neo4j 基础设施成本更高，回报为零

**发散方向**：

```
知识变现的可能性空间：

1. 订阅模式
   ├── 用户发布公开 Wiki 页面，读者订阅
   ├── 类似 Substack，但内容是编译后的知识
   └── 例："AI 论文每周解读"Wiki → 读者 $5/月订阅

2. API 计费模式
   ├── 将个人 Wiki 暴露为 MCP/REST API
   ├── 他人调用时按 token/次计费
   └── 例：某开发者的"React 设计模式"Wiki → API $0.001/次

3. 知识 NFT（激进但可行）
   ├── 将高价值 synthesis 页面铸造为可验证凭证
   ├── 证明"你是某个知识领域的首个编译者"
   └── 例：首个完整编译某论文系列的 Wiki 获得数字凭证

4. 企业知识库 SaaS（最现实）
   ├── 单用户版保持开源免费
   ├── 团队版提供权限/协作/审计 = 收费
   └── 例：类似 Obsidian Publish + AI 自动生成

5. 数据飞轮分成
   ├── 用户同意匿名贡献知识图谱数据
   ├── 用于训练行业特定模型，用户获得分成
   └── 例：1000 个医学 Wiki 用户共同训练医学知识模型
```

### 1.4 空白区 B 深度：游戏化机制

**核心问题**：知识管理是反人性的——它需要持续投入，但回报延迟且不确定。

**竞品盲区**：所有 Wiki 工具都是"严肃的"，没有利用行为心理学的激励机制。

**发散方向**：

```
游戏化层：

1. 知识探险家系统
   ├── 每日随机推荐一个"未探索"的 wiki 页面
   ├── 阅读后获得"发现点数"
   ├── 连续 7 天探索解锁"求知者"徽章

2. 链接建造师挑战
   ├── 每周挑战：在 X 和 Y 两个看似无关的概念间建立有效链接
   ├── 社区投票最佳链接
   ├──  winner 的链接被永久标记为"黄金链接"

3. 矛盾猎人
   ├── 发现两个 wiki 页面中的矛盾声明
   ├── 提交矛盾报告获得点数
   ├── 被采纳的矛盾解决获得双倍奖励

4. 知识花园成长可视化
   ├── wiki 比喻为花园：页面 = 植物，链接 = 藤蔓
   ├── 长时间不更新的页面"枯萎"（视觉变化）
   ├── 高链接密度的区域"开花"
   └── 这不是隐喻——是真实的 3D 花园可视化

5. 排行榜（可选隐私模式）
   ├── 周度"最多产的知识贡献者"
   ├── 月度"最佳链接发现者"
   └── 可以匿名参与
```

## 第二轮：跨界启发——从相邻领域偷师

> 目标：从游戏、音乐、社交、生物、建筑等领域寻找可迁移的模式

### 2.1 游戏设计 → 知识系统

**核心洞察**：最好的游戏让学习变得不可抗拒。Wiki 应该像《塞尔达传说》的神庙——每个页面是一个谜题，链接是通往新区域的门。

| 游戏机制 | 知识系统对应 | 实施思路 |
|---------|------------|---------|
| **开放世界探索** | 自由浏览知识图谱 | 首次访问某节点时播放"发现"动画 |
| **任务系统** | 知识目标 | "本周任务：建立 3 条跨领域链接" |
| **技能树** | 概念依赖图 | 显示"要理解 Transformer，需先理解 Attention" |
| **成就系统** | 知识徽章 | "图论专家"（创建 10 个 graph 相关页面） |
| **速通挑战** | 知识竞速 | "从 '量子计算' 到 ' baking' 的最短路径挑战" |
| **模组社区** | Wiki 模板市场 | 用户分享 ingest prompt 模板 |
| **新游戏+** | 知识复盘 | 每年"重玩"——用新视角重新 ingest 旧资料 |

**最激进想法**：**Roguelike Wiki**
- 每次"运行"（ingest session）随机抽取一组 raw 文件
- LLM 作为 DM（地下城主），生成独特的知识叙事
- 每次运行产出不同的概念连接，鼓励多次探索同一资料

### 2.2 音乐创作 → 知识编排

**核心洞察**：DJ 混音是将已有音轨重组为新体验。知识管理也应该有"混音"能力。

```
知识混音台（Wiki DJ Deck）：

┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  Deck A     │    │   Mixer     │    │  Deck B     │
│  来源页面    │◄──►│  概念融合    │◄──►│  来源页面    │
│  (论文 A)    │    │  (LLM 合成)  │    │  (论文 B)    │
└─────────────┘    └──────┬──────┘    └─────────────┘
                          │
                    ┌─────┴─────┐
                    │  Remix    │
                    │  合成页面  │
                    │  (新知识) │
                    └───────────┘
```

- **Deck A/B**：选择两个知识源
- **Crossfader**：调节两个来源的影响权重
- **FX 面板**：添加"对比视角"、"历史演变"、"反方观点"等效果
- **输出**：生成全新的 synthesis 页面，明确标注 remix 来源

### 2.3 社交网络 → 知识关系

**核心洞察**：Facebook 的社交图谱证明了关系数据的价值。Wiki 的链接图谱可以借鉴社交网络的发现机制。

| 社交网络功能 | 知识系统对应 |
|------------|------------|
| "你可能认识的人" | **"你可能想了解的"** — 基于浏览历史的概念推荐 |
| 共同好友 | **共同引用** — "同时引用 X 和 Y 的页面" |
| 好友动态 | **知识动态** — "你关注的概念有新的 ingest" |
| 群组 | **知识圈子** — 围绕特定主题的用户 Wiki 联邦 |
| 点赞 | **认可链接** — 标记某条 wikilink 为"高价值连接" |
| 分享 | **知识引用** — 将 wiki 页面嵌入外部文档 |
| Stories | **知识故事** — 按时间线展示某概念的演变（24h 消失） |

**最激进想法**：**Tinder for Concepts**
- 系统推荐两个看似无关的概念
- 用户滑动：右滑 = "它们有关联"，左滑 = "无关"
- 右滑后，LLM 尝试建立连接并生成解释
- 社区累积的"匹配"数据训练概念关联模型

### 2.4 生物系统 → 知识生态

**核心洞察**：生物进化是最高效的知识管理系统——DNA 是编译后的知识，变异是自然的发散思维，选择压力是质量评估。

```
Wiki 作为生态系统：

页面 = 生物个体
链接 = 食物链/共生关系
injest = 新物种引入
lint = 自然选择（淘汰缺陷个体）
graph = 生态网络
contradiction = 种间竞争
synthesis = 杂交育种
```

**具体机制**：

1. **知识突变**
   - 定期对陈旧页面进行 LLM "变异"——保留核心观点，更新表达方式
   - 用户选择"接受突变"或"保留原种"

2. **适者生存**
   - 长时间不被访问的页面进入"濒危"状态（视觉提示）
   - 如果连续 3 次 lint 都发现问题，页面进入"灭绝"（归档）
   - 用户可以通过"保护行动"（更新内容）拯救濒危页面

3. **知识共生**
   - 两个经常一起被查询的页面自动建议"共生链接"
   - 形成稳定的"知识群落"

4. **入侵物种检测**
   - 新 ingest 的页面如果与现有知识严重冲突 = "入侵物种"
   - 系统标记需要人工审核

### 2.5 建筑设计 → 知识空间

**核心洞察**：建筑师设计空间体验，不是堆砌房间。Wiki 应该设计"知识空间体验"，不是堆砌页面。

| 建筑概念 | 知识系统应用 |
|---------|------------|
| 空间序列 | 阅读路径设计——从入口页面到深度内容的引导序列 |
| 采光 | 高价值内容应该有"聚光灯"效果 |
| 尺度 | 概览页面（大尺度）→ 详情页面（人体尺度） |
| 材料 | 不同类型的知识用不同的"材质"视觉表达 |
| 废墟 | 归档页面变成"知识废墟"——可探索但标注为历史 |
| 地标 | 最重要的概念成为"知识地标"——导航中心 |

**最激进想法**：**Wiki 作为可漫游的 3D 建筑**
- 每个概念是一个房间
- 链接是门和走廊
- 大型概念是广场，小型概念是角落
- 用户以第一人称漫游知识建筑
- VR 模式下体验最佳

## 第三轮：极端场景——边缘用户与极限条件

> 目标：探索在极端条件下（无网络、无电力、大规模、匿名等）Wiki 如何工作

### 3.1 场景一：完全离线（无网络、无云端 LLM）

**问题**：所有竞品都依赖云端 LLM API。如果用户断网、API 被封、或者国家防火墙阻断，Wiki 系统如何工作？

**发散方向**：

```
离线优先架构：

Layer 1: 本地小模型（必须）
├── Ollama / llama.cpp 运行 3B–8B 模型
├── 功能降级矩阵：
│   ├── 健康检查 ✅（无需 LLM）
│   ├── 简单摘要 ✅（3B 模型可胜任）
│   ├── 实体提取 ⚠️（质量下降但可用）
│   ├── 矛盾检测 ❌（需要推理能力）
│   └── 复杂综合 ❌（必须云端）
└── UI 显示当前"离线能力等级"

Layer 2: 间歇同步（有网时批量处理）
├── 离线时操作入队（文件放入 raw/、笔记写入 wiki/）
├── 恢复网络后批量调用云端 LLM
├── 使用 CRDT 解决离线期间的冲突
└── 类似 Git 的离线工作流

Layer 3: P2P 知识联邦（无中心服务器）
├── 局域网内设备间直接同步 wiki/
├── 使用 IPFS / Hypercore 做去中心化存储
└── 每个设备是完整节点，可选性加入/退出联邦
```

**最激进想法**：**纸笔 Wiki**
- 系统生成打印友好的"知识卡片"（索引卡大小）
- 用户用实体卡片手工组织知识
- 手机摄像头拍照 → OCR → 自动更新数字 wiki
- 桥接模拟与数字知识管理

### 3.2 场景二：超大规模（100万+ 页面）

**问题**：当前所有实现都是面向 <1000 页面的。如果某企业的知识库增长到百万级页面，现有架构全部崩溃。

**发散方向**：

```
超大规模架构：

1. 分层存储
   ├── 热数据（最近 30 天访问）：内存 + SQLite
   ├── 温数据（30-365 天）：本地 SSD
   ├── 冷数据（>1 年）：对象存储（S3/MinIO）
   └── 冻结数据（>3 年）：归档压缩，按需解冻

2. 增量图谱构建
   ├── 不一次性构建全图
   ├── 按需构建子图（从查询节点出发的 N-hop）
   ├── 使用图数据库（Neo4j / Dgraph）替代 networkx
   └── 社区检测分布式执行

3. 查询路由
   ├── 简单查询 → 本地索引直接回答
   ├── 中等查询 → 检索 Top-K 页面 + LLM 综合
   ├── 复杂查询 → MapReduce 式分布式处理
   └── 全局查询 → 采样统计 + 近似回答

4. 智能预加载
   ├── 根据用户行为模式预测下一步阅读
   ├── 预加载相关页面到内存
   └── 类似操作系统的页面置换算法
```

### 3.3 场景三：匿名与加密知识库

**问题**：记者、异见人士、研究者需要管理敏感知识，但现有 Wiki 系统不提供隐私保护。

**发散方向**：

```
隐私优先 Wiki：

1. 本地加密
   ├── 整个 wiki/ 目录用 VeraCrypt / LUKS 加密
   ├── 内存中解密，不在磁盘留明文痕迹
   └── 锁屏后自动卸载加密卷

2. 匿名发布
   ├── 通过 Tor / I2P 发布公开 wiki 页面
   ├── 使用零知识证明验证内容真实性（不暴露身份）
   └── 类似 SecureDrop 的匿名提交

3.  plausible deniability（合理否认）
   ├── 双层加密：外层是"无害"内容，内层是敏感内容
   ├── 被迫交出密码时只暴露外层
   └── 类似 VeraCrypt 隐藏卷

4. 自毁机制
   ├── 连续 N 次错误密码 → 安全擦除
   ├── 紧急"恐慌密码" → 立即擦除
   └── 定时未登录 → 自动衰减（非立即销毁）
```

### 3.4 场景四：灾难恢复与长期保存

**问题**：知识应该能存活比创建它的文明更久。当前 Wiki 依赖特定技术栈，50 年后可能无法读取。

**发散方向**：

```
千年知识保存：

1. 格式永恒化
   ├── 核心知识以纯文本 + 简单 Markdown 保存（最大兼容性）
   ├── 每年自动生成"人类可读"的静态 HTML 快照
   ├── 打印为纸质版（档案馆标准纸张，200+ 年寿命）
   └── 刻录为 M-DISC（1000 年寿命光盘）

2. 自文档化系统
   ├── 系统包含自身的完整文档（如何读取这些文件）
   ├── 即使所有软件丢失，人类仍能手动解析
   └── 类似 Voyager Golden Record 的"如何播放我"说明书

3. 多副本策略
   ├── 本地副本 + 云端副本 + 朋友副本 + 图书馆副本
   ├── 使用分布式哈希表确保任何单点失效不丢数据
   └── 类似 LOCKSS（Lots of Copies Keep Stuff Safe）

4. 格式迁移机器人
   ├── 每 5 年自动检测过时格式
   ├── 自动迁移到新格式，保留原始版本
   └── 记录迁移日志（数字考古线索）
```

### 3.5 场景五：极端用户（儿童 / 老人 / 残障 / 文盲）

**问题**：所有 Wiki 工具都假设用户是受过教育的成年人。如何让知识管理普及到所有人？

**发散方向**：

```
全民知识系统：

1. 语音优先界面
   ├── "嘿 Wiki，我刚才读了一篇关于 X 的论文"
   ├── 语音转文本 → 自动 ingest → 语音确认
   └── 查询也用语音："Wiki，告诉我 Y 和 Z 的关系"

2. 图标/图像化知识
   ├── 为每个概念生成独特图标（类似古埃及象形文字）
   ├── 不识字的人也能通过图标导航
   └── 儿童版：知识作为卡通故事

3. 多感官知识
   ├── 视觉：图谱 + 颜色编码
   ├── 听觉：概念有不同音调（高音 = 抽象，低音 = 具体）
   ├── 触觉：盲文输出 / 震动模式
   └── 动觉：VR 中用手"抓取"知识节点

4. 认知辅助
   ├── ADHD 模式：时间限制阅读 + 游戏化奖励
   ├── 阅读障碍模式：OpenDyslexic 字体 + 语音同步
   ├── 记忆障碍模式：自动复习间隔（Anki 集成）
   └── 自闭症模式：结构化、可预测的界面
```

## 第四轮：技术前瞻——后 LLM 时代的知识系统

> 目标：假设当前 LLM 范式被颠覆，知识系统如何进化

### 4.1 假设一：世界模型（World Models）成熟

**背景**：未来的 AI 不再只是文本预测，而是拥有内部世界模拟能力（类似人类心智模型）。

**影响**：

```
传统 LLM Wiki：
文本 → LLM 理解 → 文本输出
（所有知识都是文本的）

世界模型 Wiki：
多模态输入 → 世界模型模拟 → 因果推理输出
（知识是关于"世界如何运作"的）

变化：
1. 页面不再只是文本，而是"世界状态片段"
   ├── 论文不再只是摘要，而是可执行的因果模型
   ├── "温室效应"页面包含可运行的气候模拟
   └── 用户调整参数，实时看到结果

2. 矛盾检测升级
   ├── 不再只是文本层面的矛盾
   ├── 而是"这两个模型在同一个世界状态下产生冲突的预测"
   └── 例：模型 A 预测利率上升→通胀下降；模型 B 预测相反

3. 知识验证
   ├── 在模拟世界中运行实验验证声明
   ├── "这个药物理论在细胞模拟中的预测是..."
   └── 知识从"被引用"升级为"被验证"
```

### 4.2 假设二：神经符号 AI（Neuro-Symbolic）

**背景**：神经网络（模式识别）与符号推理（逻辑规则）融合。

**影响**：

```
当前：LLM 提取实体和关系 → 人类验证 → 存入 wiki
未来：神经符号系统自动维护形式化知识库

1. 自动形式化
   ├── 自然语言声明自动转换为逻辑表达式
   ├── "所有鸟都会飞" → ∀x(Bird(x) → CanFly(x))
   └── 例外自动记录："企鹅是鸟，但不会飞"

2. 自动推理
   ├── 用户查询触发自动定理证明
   ├── "如果 A 则 B，如果 B 则 C，那么 A 是否导致 C？"
   └── 系统给出证明路径 + 置信度

3. 知识编译
   ├── 形式化知识可编译为可执行代码
   ├── "如果温度>30°C 且湿度>80% → 开启空调"
   └── 直接部署为 IoT 规则
```

### 4.3 假设三：具身智能（Embodied AI）

**背景**：AI 不再只是屏幕上的文字，而是物理世界中的实体（机器人、无人机、自动驾驶）。

**影响**：

```
具身 Wiki：

1. 空间锚定知识
   ├── 知识不再只是抽象概念，而是与物理位置绑定
   ├── "这个化学反应的最佳条件"锚定在实验室 3B 房间
   └── AR 眼镜在正确位置自动显示相关知识

2. 操作知识
   ├── Wiki 包含可执行的操作序列
   ├── "如何更换轮胎"不是文本，而是机器人可执行的动作计划
   └── 人形机器人读取 wiki 后执行维修任务

3. 经验知识
   ├── 机器人每次执行任务后更新 wiki
   ├── "路线 A 比路线 B 快 15%，但路面更颠簸"
   └── 知识从人类编写变为机器经验积累
```

### 4.4 假设四：脑机接口（BCI）普及

**背景**：Neuralink 等 BCI 技术让大脑直接与计算机交互。

**影响**：

```
思维 Wiki：

1. 思维摄入
   ├── 阅读论文时，BCI 直接捕获理解过程
   ├── 不需要手动写笔记——思考即记录
   └── 但隐私问题极端严重

2. 直接知识注入
   ├── 查询 Wiki 不需要阅读——直接"知道"
   ├── 类似《黑客帝国》的技能下载
   └── 但知识可能缺乏深度理解（"知道"≠"理解"）

3. 集体意识风险
   ├── 如果所有用户共享同一 Wiki，个体差异消失
   ├── "共识"变成"同质化"
   └── 需要设计"认知防火墙"保护个体思维独特性
```

### 4.5 假设五：量子计算实用化

**背景**：量子计算机解决经典计算机无法解决的问题。

**影响**：

```
量子 Wiki：

1. 量子知识图谱
   ├── 节点可以处于"叠加态"——同时是 A 和 B
   ├── "薛定谔的概念"：在进一步观察前不确定分类
   └── 更自然地表示模糊知识

2. 量子搜索
   ├── 在指数级大的知识空间中瞬时搜索
   ├── Grover 算法加速：百万页面搜索从 O(N) 到 O(√N)
   └── 超大规模 Wiki 成为可能

3. 量子加密知识
   ├── 量子密钥分发保护敏感知识
   ├── 任何窃听尝试都会破坏量子态而被发现
   └── 绝对安全的知识共享
```

## 第五轮：元设计——知识系统的终极形态

> 目标：回到第一性原理，设计知识管理的终极答案

### 5.1 第一性原理拆解

```
"知识管理"的本质是什么？

不是：
❌ 存储文件
❌ 做笔记
❌ 建数据库

而是：
✅ 扩展人类认知能力
✅ 对抗遗忘
✅ 发现连接
✅ 传承智慧

所以终极知识系统应该回答：
"如何让一个大脑 + 这个系统 > 两个大脑？"
```

### 5.2 终极形态：认知共生体（Cognitive Symbiont）

```
当前状态：工具（人使用 Wiki）
    │
    ▼
中期状态：伙伴（人与 Wiki 协作）
    │
    ▼
终极状态：共生（人-Wiki 边界模糊）

认知共生体的特征：

1. 双向学习
   ├── 人教 Wiki：ingest、纠正、评价
   └── Wiki 教人：推荐、提醒、质疑、联想

2. 共同进化
   ├── 用户使用 Wiki → Wiki 变得更适合该用户
   ├── Wiki 适应用户 → 用户思维被 Wiki 塑造
   └── 长期后：无法区分"用户本来的想法"和"Wiki 的影响"

3. 认知外包
   ├── 记忆外包：Wiki 记住一切，人只记"去哪里找"
   ├── 计算外包：Wiki 做逻辑推理，人做价值判断
   └── 创造外包：Wiki 生成候选，人做选择

4. 认知增强
   ├── Wiki 不是替代思考，而是增强思考
   ├── 类似望远镜增强视力——你看到更多，但仍是你在看
   └── 关键：人保持"元认知"——对思考的思考
```

### 5.3 终极架构：五层认知金字塔

```
        ┌─────────────┐
        │  Layer 5    │  智慧层 (Wisdom)
        │  "为什么"    │  价值判断、伦理框架、人生意义
        │  人类独占    │
        ├─────────────┤
        │  Layer 4    │  知识层 (Knowledge)
        │  "怎么做"    │  概念网络、因果关系、方法论
        │  人机协作    │
        ├─────────────┤
        │  Layer 3    │  信息层 (Information)
        │  "是什么"    │  事实、数据、定义
        │  AI 主导     │
        ├─────────────┤
        │  Layer 2    │  数据层 (Data)
        │  原始信号    │  文档、观测、测量
        │  机器处理    │
        ├─────────────┤
        │  Layer 1    │  信号层 (Signal)
        │  物理存在    │  比特、神经冲动、声波
        │  基础设施    │
        └─────────────┘

LLM Wiki Agent 当前覆盖：Layer 2 → Layer 3（部分）
终极目标：Layer 2 → Layer 4，Layer 5 永远由人类保留
```

### 5.4 终极功能：认知时间旅行

```
当前：你可以查询"现在 Wiki 知道什么"
中期：你可以查询"上周三 Wiki 知道什么"
终极：你可以查询"如果我 3 个月前做了不同决定，Wiki 会是什么样子"

实现方式：
1. 分支现实（Branching Realities）
   ├── 每次重大决策创建 wiki 分支
   ├── 可以同时探索"选择了 A"和"选择了 B"的知识后果
   └── 类似 Git 分支，但用于人生决策

2. 反事实查询（Counterfactual Queries）
   ├── "如果我没有读那篇论文，我的知识图谱会如何？"
   ├── 系统模拟"删除某知识"后的级联影响
   └── 帮助用户理解某知识的价值

3. 认知考古（Cognitive Archaeology）
   ├── "我最初是怎么相信 X 的？"
   ├── 追踪某信念的完整形成路径
   ├── 显示：原始资料 → 第一次 ingest → 后续强化 → 当前状态
   └── 发现"我早就该质疑这个信念了"
```

### 5.5 终极伦理：认知主权宣言

```
任何知识系统都必须保障的不可剥夺权利：

1. 遗忘权
   ├── 用户有权要求系统彻底删除某知识
   ├── 不是标记为删除——是彻底擦除
   └── 包括所有衍生页面中的引用

2. 误导权
   ├── 用户有权在系统中存储"故意错误"的信息
   ├── 为什么？保护性伪装、创作需要、测试系统
   └── 系统必须尊重用户的"不实之词"

3. 混乱权
   ├── 用户有权保持知识"不整理"
   ├── 不是所有知识都需要结构化
   └── 混乱有时是创造力的前提

4. 退出权
   ├── 用户可以随时完全导出所有数据
   ├── 格式必须是开放标准（Markdown、JSON、RDF）
   └── 退出后系统不得保留任何副本

5. 透明权
   ├── 系统必须解释"为什么推荐这个"
   ├── 算法逻辑可审计
   └── LLM 推理过程可追溯（Chain-of-Thought 存档）
```

### 5.6 终极商业模式：知识公地

```
不是 SaaS，不是开源捐赠，而是：

知识公地（Knowledge Commons）

1. 贡献即投资
   ├── 用户贡献知识 = 获得"知识股份"
   ├── 股份比例 = 贡献的知识被引用的次数
   └── 不是金钱，是声誉和影响力

2. 使用即认可
   ├── 查询某知识时自动"点赞"原作者
   ├── 形成去中心化的知识信用评级
   └── 最好的知识生产者自然浮现

3. 协议而非平台
   ├── 不依赖单一公司或服务器
   ├── 开放协议让任何实现都能互联
   └── 类似 Email 协议——Gmail 和 Outlook 互通

4. 长期托管基金
   ├── 每笔交易抽极小比例进入基金
   ├── 基金用于确保知识 100 年后仍可访问
   └── 类似互联网档案馆的永续机制
```

## 七、终极问题：这个项目应该成为什么？

五轮发散后，一个问题浮现：

> **"LLM Wiki Agent 是一个工具，一个平台，还是一个新物种？"**

| 定位 | 思维 | 终局 |
|------|------|------|
| **工具** | "让用户更好地管理知识" | 被更好的工具取代 |
| **平台** | "让 Agent 和人类在同一知识层协作" | 成为行业标准 |
| **新物种** | "人-Wiki 认知共生体" | 重新定义知识本身 |

当前三份原始文档都假设了**工具**定位。
`competitor-analysis-and-roadmap.md` 开始触及**平台**定位（MCP Server、联邦）。
本报告的五轮发散指向**新物种**定位。

**建议**：
- **现在**：保持工具定位，确保基础功能扎实
- **6 个月后**：向平台定位过渡（MCP 生态、联邦、API 经济）
- **2 年后**：探索新物种定位（认知共生、时间旅行、分支现实）

因为最持久的竞争优势不是功能，而是**范式**——当你重新定义了"知识管理是什么"，竞争对手就只能跟随。
