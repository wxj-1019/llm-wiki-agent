哲学一致性 · Philosophy

6 / 10 Functional

Apple-inspired 方向明确：glass morphism、rounded-xl 卡片、SF Pro 字体栈、ambient blob 背景。但系统混入两套调色板——自定义 apple-green 与 Tailwind emerald-500 在 toast 和健康评分中并存。按钮系统未收敛：apple-button（pill）、apple-button-warm（弹性缩放）、apple-button-ghost、btn-primary/btn-secondary 及大量内联样式同时存在，导致同一操作在不同页面呈现为 3–4 种不同控件。

Fix：将所有语义颜色映射到 apple-* token；合并按钮为 3 个变体（Solid / Ghost / Warm），删除 btn-primary。

视觉层级 · Hierarchy

7 / 10 Strong

字体层级清晰：text-3xl 页面标题 → text-lg 节标题 → text-sm 正文 → text-xs 元数据。Dashboard 的 section header 模式（icon + uppercase tracking-wider）一致且有效。HomePage 的搜索栏作为首要行动点，视觉权重正确。但跨页同类元素尺寸漂移：Dashboard stat cards 用 p-5 text-2xl，HomePage stat cards 用 p-4 text-3xl font-bold。"Knowledge Graph Mini" 标题使用 font-bold 而其他 section 用 font-semibold。

Fix：统一 stat card 组件，消除跨页尺寸/字重差异；建立 section title 的单一组件。

细节执行 · Detail

6 / 10 Functional

入场动画高度一致（opacity:0, y:8，duration 0.3s），glass 效果、自定义滚动条、selection 高亮均为加分项。但圆角混用：rounded-xl（卡片）、rounded-2xl（搜索/empty state）、rounded-lg（badge）同时出现。Skeleton 的雷达图占位符使用 rounded-full（圆形）而非与真实图表匹配的五边形。Sidebar 的分隔符在展开/折叠态使用两种不同样式（border-top vs 短横线）。Toast 的 progress bar 背景 bg-black/10 在 dark mode 下缺乏覆盖。

Fix：将圆角规范收敛到 rounded-xl（16px）与 rounded-full（pill）两级；修正 Skeleton 雷达图形状；为 progress bar 添加 dark mode track 色。

功能性 · Functionality

8 / 10 Strong

防御性工程做得扎实：vim 风格键盘快捷键（g h/b/g）、焦点陷阱、ARIA 标签（role="alert"、aria-live="polite"）、响应式断点（md:）、骨架屏 loading、通知系统的 progress toast。页面切换的 AnimatePresence 使导航有明确的流向感。唯一瑕疵：ToastContainer 的固定宽度 w-80 即使无 toast 时也会阻挡右侧 320px 的点击穿透；Sidebar 的 nav item py-2 在移动端 drawer 中 touch target 偏小。

Quick win：将 ToastContainer 改为仅在 toast 存在时渲染，或将其宽度收缩为 toast 实际宽度。

创新性 · Innovation

5 / 10 Functional

整体处于"安全但平庸"的区间。Apple 美学本身是成熟设计系统，但作为"AI 驱动的知识管理工具"，视觉语言并未传达出智能、联结或认知的独特气质。GrowthTrendChart 的 CSS stroke-dashoffset 动画和 ambient blob 背景是不错的 touches，但缺乏一个令人难忘的视觉签名。没有 unexpected 的排版手势、没有专属的布局 motif、没有 material treatment 能让人感到"这是 Wiki Agent，不是又一个 Apple Dashboard"。

Fix：引入一个差异化的视觉符号——例如将知识图谱的边线动态映射到页面背景中，或使用节点脉冲动画作为 loading indicator，让"图"的概念渗透到 Chrome 层。

✓ Keep · 保持

  • ✓ CSS 变量系统：light/dark 双主题通过 :root 与 [data-theme='dark'] 切换，扩展性好。
  • ✓ 入场动画节奏：统一的 y:8 + opacity 0.3s 入场，使全站有连贯的呼吸感。
  • ✓ glass 卡片：.glass 的 backdrop-blur + 半透明边框质感干净，不要丢。
  • ✓ 键盘快捷键：vim 和弦导航是产品的差异化功能，保持可见性。
  • ✓ 图标尺寸梯度：14px（按钮）→ 16px（nav）→ 18px（header）→ 20px（stats）的体系合理。

✕ Fix · 必须修复

  • ✕ 统一按钮系统：删除 btn-primary/btn-secondary，收敛为 3 个变体。
  • ✕ 消除双调色板：将 emerald-500、red-500 替换为 apple-green、apple-red。
  • ✕ 统一 stat card：HomePage 与 Dashboard 的统计卡片尺寸/字重必须一致。
  • ✕ 圆角规范：卡片/容器统一 rounded-xl（16px），交互元素（button/badge）用 rounded-full。
  • ✕ Toast 占位：空状态时不应占据 320px 点击盲区。
  • ✕ Skeleton 诚实性：雷达图占位符应为五边形而非圆形。

★ Quick Wins · 快速收益

  • ★ 标题字重：全局搜索 font-bold 替换为 font-semibold（仅保留数据高亮）。
  • ★ Progress bar dark：为 toast progress track 添加 dark:bg-white/10。
  • ★ Sidebar 分隔符：collapsed 态的短横线改为与 expanded 态一致的 border-t 逻辑。
  • ★ Empty state 圆角：将 empty state 卡片从 rounded-2xl 降为 rounded-xl 以匹配全局。
  • ★ Search focus ring：将 shadow-[0_0_0_3px_rgba(10,132,255,0.15)] 提取为 --focus-ring token。

下一步：从"Apple Clone"走向"Wiki Agent 身份"

当前设计是安全的、可读的、专业的——但它可以属于任何 SaaS Dashboard。作为 AI 驱动的知识图谱工具，视觉语言需要开始回答"为什么是这个产品"。以下是基于 frontend-design skill 的三个差异化方向：

  • Organic + Neural（推荐）：保留现有 rounded corners 和温暖色调，引入微妙的节点网络纹理（SVG feTurbulence 粒子）作为页面背景，让"知识图谱"的隐喻渗透到 chrome 层。配色从纯 Apple 转向有机 earth tones（sage, clay, moss），传达知识的生长感而非工具的冰冷感。
  • Swiss + Graph Paper：将当前设计推向更极端的 Swiss 方向——纯白背景、可见的 1px 网格线、不对称排版、将节点/边的数值作为构图元素（类似印刷品中的 folio number）。适合强调"结构化知识"的精确感。
  • Retro-Futuristic Terminal：深色 phosphor green + amber 双色调，CRT scanline 纹理，等宽字体主导。适合极客用户群体，但会牺牲目前的通用亲和力。

建议先完成 Fix 列表中的系统级收敛，再选择一个差异化方向进行下一轮的视觉跃迁。
