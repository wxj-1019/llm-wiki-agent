Critique · LLM Wiki Viewer

Reviewer: OD · Critique skill  |  2026-05-05
A solid Apple-style knowledge dashboard with strong functional bones and emerging polish — hierarchy holds on desktop but frays on mobile; innovation potential is untapped.
6/ 10  ·  Functional

Philosophy consistency

The Apple-style direction is declared and mostly followed — capsule buttons, rounded-xl cards, blue accent (#0a84ff), SF Pro system font. But drift appears: ChatPage user bubbles use mint-green (A7F3D0) which clashes with the cool-blue thesis; GraphPage vis.js nodes use a separate hardcoded palette that ignores the design system; StatusPage panel headers mix purple/violet/pink/blue without a consistent semantic rule.

  • Keep: The capsule button + card vocabulary is coherent across 80% of pages.
  • Fix: Unify Graph/Mindmap node colors with CSS variables (recently fixed).
  • Quick win: Replace ChatPage mint bubble with a neutral/blue tint.
7/ 10  ·  Strong

Visual hierarchy

Desktop hierarchy is clear: fixed header (z-50) → sidebar navigation → page title (text-3xl) → content grid. But mobile hierarchy frays: the onboarding overlay on GraphPage blocks the entire viewport without a clear dismiss affordance; PageDetailPage edit toolbar crowds the header on narrow screens; DashboardPage stat cards and charts compete for attention without size contrast.

  • Keep: Sidebar active state (border-l-2 + bg tint) provides excellent wayfinding.
  • Fix: Graph onboarding should be a compact tooltip, not a full-screen modal.
  • Quick win: Reduce DashboardPage chart titles from text-lg to text-sm to de-emphasize.
7/ 10  ·  Strong

Detail execution

Page transitions (AnimatePresence) and card hover shadows are polished. The signal-strength bars on BrowsePage cards are a delightful micro-detail. But execution gaps remain: SettingsPage tab switch has no motion; TimelinePage event dots use bg-opacity-10 which is nearly invisible; ChatMessage streaming state shows a flash of "No sources" before sources arrive; reading progress bar on PageDetailPage takes up fixed space even at 0%.

  • Keep: Signal-strength bars and page fade transitions show craft.
  • Fix: SettingsPage tabs need a sliding indicator or cross-fade.
  • Quick win: Conditionally render reading progress bar only when > 0.
8/ 10  ·  Strong

Functionality

Robust keyboard navigation (CommandPalette Cmd+K, "/" shortcut to focus chat, Escape handlers), focus traps on all dropdowns, skip link for screen readers, PWA offline banners, and responsive breakpoints (md/lg/xl). The recent accessibility sweep fixed 30+ aria-label issues. Remaining gaps: GraphPage edit mode is undiscoverable (no tooltip on the toggle); ChatPage session list lacks search/filter when sessions grow; no print styles for wiki pages.

  • Keep: Focus traps + skip link + keyboard shortcuts are production-grade.
  • Fix: Add a tooltip or label to GraphPage "Edit mode" toggle.
  • Quick win: Add `@media print` styles to hide sidebar/header and show full content.
6/ 10  ·  Functional

Innovation

The knowledge graph visualization and signal-strength backlink indicators are the two memorable moments. Chat-with-wiki-context is a practical integration. But the app stays in safe territory: no micro-interactions (drag-to-reorder favorites, haptic feedback on save), no ambient intelligence (suggest related pages while reading), and the graph interaction is standard vis.js pan/zoom. There is room for one unexpected move that would make users lean in.

  • Keep: Signal-strength cards and graph are genuinely useful innovations.
  • Fix: Consider a "related pages" inline suggestion panel on PageDetailPage.
  • Quick win: Add a satisfying toast animation (slide + scale) instead of plain fade.

Keep

  • Capsule button + rounded card vocabulary — it is the app's recognizable silhouette.
  • Sidebar active state (border-l-2 + bg-apple-blue/10) — best-in-class wayfinding.
  • Page-level AnimatePresence transitions — they make navigation feel expensive.
  • Signal-strength backlink bars on BrowsePage cards — a delightful data-visualization micro-moment.
  • Keyboard shortcuts (Cmd+K, "/", Escape) — defensively engineered for power users.

Fix

  • GraphPage onboarding overlay is too heavy — replace with a compact 3-step tooltip tour. P0
  • ChatPage mint-green user bubble clashes with the cool-blue design thesis. P0
  • SettingsPage tabs need a sliding indicator or motion to reinforce selection. P1
  • GraphPage "Edit mode" toggle is undiscoverable — add a tooltip or label. P1
  • PageDetailPage reading progress bar renders at 0% wasting 2px of fixed space. P1

Quick wins

  • Add `@media print` styles to hide sidebar/header and expand content width.
  • Replace ChatMessage "No sources" flash with a skeleton dot or delayed render.
  • Add a toast entrance animation (slide + scale bounce) instead of plain x-translate.
  • TimelinePage event dots: increase bg opacity from 10% to 20% for visibility.
  • DashboardPage chart titles: reduce from text-lg to text-sm to improve hierarchy.
