# useRef

`useRef` is a foundational React hook that provides a mutable reference object persisting for the full lifetime of a component, distinct from React's standard state management. In this wiki's source code, `useRef` primarily serves two critical functions: preserving values across renders without triggering re-renders, and holding references to DOM elements or imperative handles. The `UseCountUpHook.md` and `usecountup-animated-counter-hook.md` sources demonstrate its use in `useCountUp.ts` where a `fromRef` captures the current displayed animation value at the moment a target change occurs, enabling smooth mid-animation transitions without the overhead of a state update that would cause visual jumps. Similarly, `UseJarvisMoodHook.md` and `usejarvis-mood-jarvis-mood-state-hook.md` reveal that `useJarvisMood` employs a `moodRef` to synchronously track the current mood state, thereby avoiding stale closures in callbacks such as timers or event handlers that might otherwise capture outdated values from a closure over the state variable. In `usefocustrap-focus-trap-hook.md`, `useFocusTrap` leverages a `ref` to reference the container DOM element and a ref to store the previously focused element, allowing focus restoration upon deactivation without causing re-renders. Across all cases, `useRef` acts as the engineer's tool for bridging React's declarative rendering model with imperative, time-sensitive, or persistence-oriented concerns — holding animation baselines, guarding state transitions, and managing focus boundaries — all without imposing the performance cost or rendering cycles that `useState` would incur. The hook returns an object with a mutable `.current` property that, once set, remains available across the component's lifecycle and is particularly valuable in animation loops, timeout callbacks, and DOM interactions where reading the latest value must happen synchronously.
